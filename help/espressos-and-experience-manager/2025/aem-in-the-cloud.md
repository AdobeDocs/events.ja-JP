# AEMがクラウドでより強力な理由

このセッションでは、AdobeのBusiness Advisor、Casey Witter、Technical AdvisorのRajan Kumarが、クラウドへの移行の基本について説明します。 クラウドへの移行により、スケーラビリティとアクセシビリティが向上し、進化する需要に迅速に適応できるようになります。 Caseyが、コンテンツとバックエンドの徹底的な計画と評価から始めて、Adobeの信頼性の高い方法論を紹介し、効率的な移行を実現するための方法を紹介します。 このプロセスでは、リソースを効果的に割り当てる詳細な移行計画が必要です。 また、導入のための主要なワークストリーム、プロセスを合理化するための有用なツール、予想されるタイムラインに関するインサイトも探っています。 続いて、Rajanが、AdobeのベストプラクティスアナライザーとCloud Acceleration Managerツールを活用して、クラウドへの準備状況を評価する方法を紹介します。 質疑応答の際には、いつものように回答させていただきます。

>[!VIDEO](https://video.tv.adobe.com/v/3443023/?learn=on&enablevpops)

## 重要な留意点

* **パフォーマンスの向上** Cloud Serviceのお客様は、より優れたアーキテクチャ、アセット処理、グローバル データセンターにより、ページの読み込み時間が400%向上しました。

* **最新イノベーションへのアクセス** Cloud Serviceでは、迅速なサイト作成、Adobe JetおよびStudio、強化されたセキュリティ、HIPAAへの準拠など、最新のコンテンツイノベーションにアクセスできます。

* **市場投入までの時間の短縮** Cloud Serviceのお客様は、自動化されたCI/CD、柔軟なデプロイメント、開発環境の改善により、市場投入までの時間が2倍短縮されました。

* **移行ツールと方法論** Adobeには、移行プロセスを評価および促進するためのベストプラクティスアナライザー（BPA）やCloud Acceleration Manager（CAM）などのツールが用意されています。 推奨されるアプローチは、段階的な手法**準備状況、計画と設定、実装、運用開始、最適化です。

* **運用の安定性とサポート** Adobeは、頻繁なバックアップ、リカバリープロセス、グローバルデータセンターなどの機能を備えたクラウドサービスの安定性を高めます。 クラウドサービスには、自動デプロイメントプロセスと強化されたセキュリティ対策も含まれています。
