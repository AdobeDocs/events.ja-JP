# Data in Motion - Insightを活用したチームのためのスケーラブルな分析ワークフロー

Adobe Adobeのシニア分析マネージャーであるIoana Maftei氏が、IBMコンテンツを測定可能なものにする、拡張性の高いガバナンス付きの分析ワークフローを構築する方法を解説します。 適切な関係者に的確なインサイトをタイミングよく提供し、部門をまたいだ共同作業を促進して、さまざまな役割やニーズに合わせてAnalysis Workspaceプロジェクトをカスタマイズする方法をご確認ください。

>[!VIDEO](https://video.tv.adobe.com/v/3471118/?learn=on&enablevpops)

## Analytics ステージへの役割のマッピング

| 分析ステージ | 関与する主な役割 | 責任 |
|--------------------------|-----------------------------------|--------------------------------------------------|
| 技術計画 | アーキテクト，タグ付けスペシャリスト，プロダクトオーナー，マーケティングリード | ビジネスニーズの翻訳、データキャプチャの計画 |
| タグ付けと実装 | 開発者、タグ付けスペシャリスト | トラッキングの実装とデプロイ |
| 検証 | web アナリスト、開発者 | データの正確性と一貫性の確保 |
| 分析/キュレーション | プロダクトオーナー、マーケティングアナリスト、web アナリスト | カスタマイズされたレポートを作成し、インサイトを獲得 |
| Collaborationとアクティベーション | すべての関係者 | インサイトの共有、活用、アクション |


## 分析の課題を克服

* **分断の解消**&#x200B;技術チームとビジネス チームを早めに調整して、共通の理解と賛同を得ます。
* **データ検証** システム間に若干の相違が生じる可能性があります。明確な定義を設定し、許容できる差分について関係者に情報を提供します。
* **導入の促進** ライブデモと役割固有のダッシュボードを使用して、信頼を築き、価値を実証します。
* **透明性** データの問題とコンテキストを積極的に伝え、特に経営陣との信頼関係を維持します。

これらの課題に対処することで、分析が真のビジネス成果を生み出し、データ主導の文化を育むことができます。
