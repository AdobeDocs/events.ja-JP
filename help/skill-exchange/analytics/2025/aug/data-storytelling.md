# Data StorytellingとAnalysis Workspace

ダッシュボードの能力は高くても、あまり多くの場合、成果につながりません。 このセッションでは、Adobe Analysis Workspaceのプロジェクトを実際のデータストーリーに変換し、注目を集め、行動を促す方法を説明します。

ビジュアライゼーションをより意図的に活用する方法、より優れた質問（AIの助けを借りて）をする方法、5 Whys手法などの手法を学び、本当に重要なことを深く掘り下げる方法などについて解説します。 最終的には、オーディエンスを「So What」に導くダッシュボードを構築し、より迅速で優れた意思決定を支援することができます。

>[!VIDEO](https://video.tv.adobe.com/v/3471116/?learn=on&enablevpops)


## 効果的なデータStorytellingを活用

従来のレポートから脱却し、関係者の行動を促進するダッシュボードを作成する方法をご確認ください。

* **Storytellingによる行動の促進**&#x200B;関係者は、生データだけでなく、insight、緊急性、感情的な関連性にも対応します。
* **ストーリー構造の問題**&#x200B;設定、張力、解像度を使用して、ユーザーをダッシュボードに誘導し、効果的なstorytellingを反映させます。
* **ビジュアルとコンテキスト**&#x200B;高度なビジュアライゼーションとコンテクストのテキストにより、ユーザーは重要なことに集中し、感情的なエンゲージメントを構築できます。
* **プレゼンテーションは重要です** データストーリーを提示する方法は、そのインパクトを生み出したり壊したりすることができます。個人配信は、感情と明瞭性を高めます。

これらの原則を適用することで、有益なだけでなく、有意義な意思決定を促すダッシュボードを作成することができます。

## 視覚的階層とナラティブ円弧

* ストーリーなどの構造ダッシュボードは、設定（主要な統計または状況）、緊張（なぜ問題が存在するのか）の導入、レコメンデーションによる解決から始まります。
* ストーリーを読むようにダッシュボード内を誘導し、各セクションでコンテクストと感情を構築します。
* セクションヘッダーを使用したり、不要なタイトルを非表示にしたり、プロフェッショナルなブランディングのために画像やロゴを追加したりします。
* ダッシュボードのフローを関係者のニーズに合わせ、ストーリーを明確かつ実行可能にする。
