# Customer Journey Analyticsのデータモデルにアプローチする方法

CJAデータモデルを構築する上で重要な側面のひとつは、異なる顧客接点やインタラクション間の階層的な関係を把握することです。 これが、有意義な分析とインサイトの基盤です。

主要な検討事項は、次のとおりです，

* あらゆるチャネルをまたいだ顧客インタラクションポイントの特定とマッピング
* 明確なイベント階層と関係の確立
* 一貫性のあるアトリビューションモデルの定義
* 標準化された指標とKPIの作成

これらの要素を適切に構築することで、カスタマージャーニー全体をより適切に追跡および分析し、より実用的なインサイトの獲得と的確な意思決定を実現できます。

>[!VIDEO](https://video.tv.adobe.com/v/3471111/?learn=on&enablevpops)

## 強力な分析のためのデータモデリング

Adobe Experience Platform（AEP）およびCustomer Journey Analytics（CJA）の効果的なデータアーキテクチャが、アクションにつながるインサイトとレポートをどのように促進するかをご確認ください。

* **スキーマ設計の重要** フラットスキーマ、配列、オブジェクトの配列の選択は、分析機能とレポートの柔軟性に直接影響します。
* **変換プロセス** AEPに取り込まれたデータは、CJAでシームレスな変換と使いやすさを確保するために、慎重に構造化する必要があります。
* **コンテナ階層** イベント、セッション、人物のレベルを理解することは、マルチレベル分析と正確なレポート作成に不可欠です。
* **実用的な戦略**&#x200B;先行計画、スキーマガバナンス、プラットフォーム機能の活用は、スケーラブルで将来を見据えた実装の鍵となります。

これらの概念を習得することで、分析ワークフローを最適化し、より深いビジネスインサイトを引き出すことができます。

## スキーマタイプとそのユースケース

* **フラットスキーマ**&#x200B;わかりやすい1対1のデータ関係に最適です。 基本的なイベント追跡とシンプルな指標/ディメンションに最適です。
* **配列**&#x200B;関連項目（製品カテゴリ、コンテンツタグなど）のリストに役立ちます。 各配列要素は、分析のための個別のディメンションになります。
* **オブジェクトの配列**&#x200B;製品の購入など、各オブジェクトが独自のプロパティと関係を維持する複雑なユースケース向けに設計されています。 詳細なオブジェクトレベルの分析が可能です。
* **賢く選ぶ** ニーズに合った最もシンプルなスキーマを選択しますが、関係の維持を必要とする高度なシナリオには、配列とオブジェクトを活用します。
