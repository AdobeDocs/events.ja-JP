# 新機能でユーザーを有効にする：興奮は人生です

「興奮は停滞の反対です。一方は生命、もう一方は死です」 ～Ernestine Rose システム管理者として、リリースノートは、システムを改善し続け、より多くの問題を解決する方法に関する次の興味と見なされるかもしれませんが、多くのユーザーにとって、変化は必ずしも簡単に受け入れられるわけではありません。 当社は、変化の文化を構築し、適切な関係者に解決策を提示し、システムを継続的に改善する方法を模索し続けることで、システムが真新しいものであれ、全速力で稼働するものであれ、継続的に価値を付加し続けています。 一般的なシステムの更新を共有するためのシステム（ガバナンス）変更の文化を構築するためのヒント – ユーザー導入を可能にする変更に最適なROIの特定 – 価値が高く、労力が少ない最初のベースラインと影響に関するレポート。

## Workfrontにおけるイノベーションと導入の促進

Workfrontの可能性を最大限に引き出すには、変革、機能の導入、継続的な改善に対する戦略的アプローチが必要です。

* **文化を変える** イノベーションを歓迎し、停滞を回避するマインドセットを醸成します。魅力的なコミュニケーションと定期的なセッションを利用して、興奮を高めます。
* **機能の調整**&#x200B;新機能をビジネス目標に結び付け、様々な関係者に合わせてメッセージをカスタマイズして、関連性と賛同を最大化します。
* **Champions &amp; Collaboration**&#x200B;特に大規模または複雑な組織で、管理者をサポートし、導入を支援するために変革担当者を特定し、育成します。
* **継続的な改善**&#x200B;分析、フィードバック、構造化コミュニケーション（SBARなど）を使用して、影響を測定し、ロールアウトプロセスを改善します。

これらのインサイトを活用することで、導入を成功させ、価値を最大化し、組織の俊敏性と競争力を維持することができます。

## 変化に向けたチームの連携

* 変化を受け入れる文化を構築するには、次のことをおこなう必要があります。
   * エネルギーに満ちた魅力的なコミュニケーションの使用（モノトーン、ドライ「リリースノート」を避ける）
   * Adobeリリースに合わせて四半期ごとに「新機能」セッションを開催
   * ニュースレター、チームハドル、タウンホールを利用して、最新情報を入手する
* チャンピオンの特定と育成、チーム内の変革担当者の育成
* ユーザーやグループ管理者が、チームと関連する最新情報を共有できるようにします
* すべてのユーザーをWorkfrontに強制するのは避け、必要に応じて他のツール（JIRA、Smartsheet）と統合します
