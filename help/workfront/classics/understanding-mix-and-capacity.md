# エキスパートに聞く – ミックスとキャパシティを理解する

企業内のミックスとキャパシティを測定する方法について説明します。 このウェビナーは2019年10月2日に録画されました。

>[!VIDEO](https://video.tv.adobe.com/v/341119/?quality=12)

## Q&amp;A

**質問**

%sを棒グラフにどのように配置しますか？

**回答**

ミックスレポートに記載されている%の値は、「グラフ」タブで「グループ列」と「100%に積み重ね」を選択したためでした。 「積み重ね」を選択した場合は、予定時間の合計が表示され、パーセントは表示されません。

**質問**

部門/組織のワークロードがプロジェクト/タスクとイシュー/リクエストの組み合わせである場合、WPIの上位レベルのレビュー（Workfront レポート）を取得することをお勧めします。

**回答**

プロジェクト、タスク、イシューには、それぞれ独自のカスタムフォームを使用した独自のレポートが必要です。 ただし、同じ作業タイプフィールドを使用することもできます。 3つのレポートを1つのダッシュボードに表示することもできます。

**質問**

「タスクを一括編集して、業務に活用したり、戦略的に活用したりする必要があるでしょうか？」

**回答**

私たちが提案している手法は、運用中のタスクのリストを取得できるレポートを作成することです。 すべてのタスクを一度に選択できるようになったら、一括編集し、作業タイプを使用してカスタムフォームを添付し、すべてのタスクに対して作業タイプを一度に「運用中」に設定します。 同じ手順に従って、戦略的タスクのリストを収集したり、一括編集したり、カスタムフォームを追加したりします。

タスク名、プロジェクト所有者、ポートフォリオ、ユーザーが割り当てられた特定の単語を確認するなど、リストの取得に役立つカスタムプロンプトに関するアイデアをいくつかウェビナーに紹介します。 これらは単なるアイデアです。 自社の状況に応じて最適なレポートを考案する必要があります。

**質問**

ミックスに4つのカテゴリがある場合、それぞれに目標を作成し、予測と計画と実際の間の差分についてレポートすることはできますか？ （4つのカテゴリのキャンペーン、事業部、Web、製品）。 プロジェクトレベルのカスタムフォームやフィールドには。 目標は、四半期ごとの予測（予算/予測）を作成し、それに向けて予定時間を追跡し、最終的には実際の時間を追跡することです。

**回答**

1つのカスタムフィールドにすべてのカテゴリがある場合（今は「作業タイプ」と呼びましょう）、最初に「予定時間」と「作業タイプ」でグループ化されたプロジェクトレポートを作成するだけです。 プロジェクトレポートをフィルタリングして、目的の日付範囲内の計画にプロジェクトを表示します。 グループ化された列または棒グラフを100%に積み重ねて使用すると、割合を確認できます。 これはあなたの予測レポートかもしれません。

レポートをコピーして編集し、現在のプロジェクトに基づくレポートを作成しても、予定時間に基づくミックスが表示されます。

レポートをもう一度コピーし、予定時間ではなく実際の時間でグループ化し、完了したプロジェクトのみを目的の日付範囲内で表示できます。 これは、実際の時間にもとづくミックスの割合を示します。

**質問**

これは、プロジェクトまたはタスクに複数のカテゴリ IDがある場合に機能しますか？

**回答**

はい、複数のIDがある場合は、次のようにタブで区切る必要があります。

```
EXISTS:1:$$EXISTSMOD=NOTEXISTS
EXISTS:1:$$OBJCODE=OBJCAT
EXISTS:1:categoryID=5d76d82600e7926bb51eeb1e0886810e 5d54288d01034619f2eb2c74f6472f18
EXISTS:1:objID=FIELD:ID
```

タブ文字で区切る最適な方法は、まずビルダーでカテゴリのリストを作成することです。 複数のカスタムフォーム名を入力し、テキストモードに切り替えると、タブで区切られたIDとして表示されます。

複数のIDはORとして扱われるので、いずれかのIDがタスクに添付されている場合、レポートには表示されません。

**質問**

レポートのプロンプトは「ANDed」ですか、「ORed」ですか。

**回答**

個々のカスタムプロンプトは「ANDed」です。 したがって、プロジェクト所有者としてPamを指定し、タスクに割り当てられたBillを指定すると、Pamがプロジェクト所有者であるプロジェクト内のBillに割り当てられたタスクのみが表示されます。

**質問**

特定の列だけでソートできるのはなぜですか？ つまり、割り当てで並べ替えることはできません。

**回答**

「割り当て」はリストであり、アイテムのリストで並べ替えたりグループ化したりすることはできません。 並べ替えまたはグループ化できるのは個々の項目のみです。

この点を説明するために、1つのタスクにこのような割り当てのリストを想像してみてください。

```
Jane
Bill
Dan
```

別のタスクでの割り当てのリストが表示されます。

```
Bill
Jane
Helen
```

どのタスクを並べ替えて最初に表示する必要がありますか？ リスト内の名前で並べ替えることはできますが、順序を制御できないので、これは必ずしも便利ではありません。 Workfrontでは、リストによる並べ替えをまったく許可しないため、この問題を回避できます。

リストでグループ化する方法は？ 名前のリストでグループ化すると、Jane、Bill、Danに割り当てられたすべてのタスクがグループ化され、Jane、Dan、Billに割り当てられたすべてのタスク（同じリストですが、異なる順序で）が異なるグループ化されます。 Workfrontでは、リストによるグループ化を許可しないため、この問題を回避できます。

**質問**

戦略的タスクに使用される予定時間と、実際の業務時間を把握していますか？

**回答**

いいえ。 この例では、予定時間を使用して、戦略的タスクと運用的タスクの両方に計画された労力レベルを示しています。 過去、現在、未来を問わず、容易に比較することができます。 また、実際の作業時間を使用して戦略的タスクと業務タスクを比較することもできますが、実際の作業時間は、実際にタスクに取り組んでいた時間として報告された時間であるため、過去のタスクに限られます。

**質問**

リソースプランナーでは、過去に計画されたものの、完了しなかったタスクをどのように考慮しますか？ 予定時間数はロールフォワードされていないようなので、リソースを必要とするタスク/時間として今後数週間は表示されません。

**回答**

未完了の作業の「自動」ロールフォワードはありません。 この場合、プロジェクトを再計画する必要があります。 たとえば、あるタスクに最初に割り当てたリソースが、新しい期間では利用できないため、プロジェクトマネージャーはこれを検証し、再計画するための最適な方法を決定する必要があります。 つまり、関係者を巻き込んで、計画の変更の承認を得る必要があります。

**質問**

メールをチェックするために2時間/日を入力するのではなく、休憩を追加することをお勧めします。

**回答**

はい、FTEを0.75に設定すると、1日6時間利用可能なユーザーがリソースプランナーに表示されます。 これは毎日の空き時間になります。 各四半期の最終日が使用不可など、日付に応じて異なる期間で使用不可として表示する場合は、間接費プロジェクトを使用します。

一部の人は、自分でプロジェクトを作成し、好きなときに変更できるため、オーバーヘッドプロジェクトを好むが、独自のFTEを編集する権利がない場合がある。

**質問**

チームキャパシティダッシュボードのデータは、作業に対する権限に関係なく、レポートを共有する誰でも利用できますか？

**回答**

ユーザーがオブジェクトを表示する権限を持っていない場合、レポート/ダッシュボード内には表示されません。 ただし、全員に同じ結果を表示する場合は、レポートアクション / 編集/ レポート設定に移動し、「このレポートをアクセス権で実行する」フィールドに自分の名前を入力します。 これにより、このレポートで共有されているユーザーは、表示される正確な結果を確認できます。 結果そのものへの追加アクセスを許可しないため、一部の結果はクリックできる場合とできない場合があります。

**質問**

タスクではなく、プロジェクト全体に割り当てられたすべてのスタッフを示すレポートを作成するにはどうすればよいですか？

**回答**

「スタッフ」タブ（プロジェクトチーム）の一部としてリストされたすべてのユーザーを表示する列をプロジェクトレポート内に作成できます。 次のテキストモードを使用します。

```
displayname=Project Team
listdelimiter=<p>
listmethod=nested(projectUsers).lists
textmode=true
type=iterate
valuefield=user:name
valueformat=HTML
```

**質問**

チームの働き方を取り入れたレポート/ダッシュボードが必要です。 特に、次のシナリオがあります。 – 自分が所有するプロジェクト / 自分のために作成されたプロジェクト / 自分が他の人に割り当てたタスク / 自分に割り当てられたタスク

**回答**

所有しているプロジェクト

「現在のプロジェクト」という名前のビルトインレポートがあり、すべての現在のプロジェクトが表示されます。 このプロジェクトを編集し、フィルタールール :Project/所有者ID/次に$$USER.IDを追加して、レポートを保存し、名前を「自分が所有するプロジェクト」に変更できます。

自分で作成したプロジェクト

「マイプロジェクト」という名前のビルトインレポートがあり、プロジェクトチームに所属しているすべての現在のプロジェクト（所有者、スポンサー、タスクに割り当てられているプロジェクト）が表示されます。 これが何を求めているのかはわかりませんが、誰かがプロジェクトのオーナー、スポンサー、またはタスクへの割り当て以外に、プロジェクトを作成したかどうかを知る方法はありません。

自分が他のユーザーに割り当てたタスク

必要なフィルターを使用してタスクレポートを作成し、「フィルター」タブに移動して、「テキストモードに切り替え」をクリックします。 このコードを既存のコードに追加します：

```
EXISTS:1:$$OBJCODE=ASSGN
EXISTS:1:taskID=FIELD:ID
EXISTS:1:assignedByID=$$USER.ID
```

これにより、ログインしたユーザーが現在の担当者の少なくとも1人を割り当てたタスクがすべて表示されます。 担当者が複数のユーザーによって割り当てられた場合、誰かを割り当てた最初のユーザーの名前のみが、タスクのランディングページに「依頼者」として表示されます。 割り当てられたすべての人物と各メンバーを割り当てた人物を表示したい場合は、ビューに列を追加し、テキストモードに切り替え、そこにあるものを次のように置き換えることができます。

```
displayname=All Assignees and Requesters
listdelimiter=<p>
listmethod=nested(assignments).lists
textmode=true
type=iterate
valueexpression=CONCAT("Assigned To: ",{assignedTo}.{name},"; Requested By: ",{assignedBy}.{name})
valueformat=HTML
```

自分に割り当てられたタスク

「マイタスク」という名前のビルトインレポートがあり、自分がタスクオーナーである現在のプロジェクトのすべての不完全なタスクが表示されます。 フィルターを変更して、タスクの所有者だけでなく、割り当てられた可能性のある多くのユーザーの1人であるすべてのタスクを表示することをお勧めします。 これは、フィルタールールを削除することで実行できます

```
Task > Assigned To ID > Equal > $$USER.ID 
```

との置き換えについて説明します

```
Assignment Users > ID > Equal > $$USER.ID
```

**質問**

グラフのラベルをカスタマイズする方法はありますか？ プロジェクトのステータスを反映するチャートを作成すると、ホームグループの名前がラベルに含まれることがわかりました。

**回答**

グラフのラベルには、グループ化する項目のフィールド名が使用されます。 ラベルを変更する唯一の方法は、名前を問わず、計算されたカスタムフィールドを使用することです。 フィールドの計算セクションに、グループ化するネイティブフィールドのフィールド名を入力します。

**質問**

Chuckのチーム割り当てのレポートバーをどのように色分けしますか？ すなわち、後ろは琥珀、遅い方は赤、時間通りに緑？ また、より論理的な順序に変更することもできます。例えば、赤/アンバー/緑または逆にすることはできますか？

**回答**

「進捗ステータス」オプションでレポートで使用する色を変更するには、レポートを編集して「グラフ」タブをクリックします。 「カスタムカラー/」ドロップダウンを探します。 列または棒をグループ化するかどうかに応じて、「左（Y）軸」ボックスまたは「データをグループ化」ボックスの横に表示されます。 このドロップダウンで色を選択できます。 カラーオプションの右下にある数字をクリックすると、大きなパレットからカラーを選択できます。

残念ながら、これらの順序を変更することはできません。

**質問**

作業者にタスクが割り当てられているプロジェクト数を指すグラフを作成できますか？

**回答**

はい、以下の方法で実行できます。

* プロジェクトレポートの作成
* 問題のユーザーがログインユーザーの場合、フィルターには次の行を含める必要があります。

```
   Project Users > ID > Equal >$$USER.ID 
```

* そうでない場合は、ユーザー名を[!DNL $$USER.ID]に置き換えます。 これにより、そのユーザーがタスクを割り当てられているか、所有者またはスポンサーであるすべてのプロジェクトが表示されます。 タスクが割り当てられているプロジェクトのみを表示する場合は、次の2つのフィルタールールを追加する必要があります。

```
   Project > Owner ID > Not Equal > $$USERID
   Project > Sponsor ID > Not Equal > $$USERID
```

* 1つ以上のグループ化を作成して、グラフを作成します。 会社など、何でもグループ化します。 次に、「グラフ」タブをクリックし、グラフを選択します。 「レコード数」は、1つの軸のデフォルトになります。 これは、ユーザーが割り当てを持つプロジェクトの数です。

ユーザーがプロジェクトの割り当て（タスク、プロジェクトオーナー、プロジェクトスポンサーに割り当て）を与えられると、そのユーザーはプロジェクトチームに追加され、「人」サブタブの「スタッフ」タブに表示されます。 ユーザーがプロジェクトオーナー、スポンサー、またはタスクの割り当てから削除された場合、そのユーザーの名前はプロジェクトチームに残ります。 削除する場合は、手動で削除する必要があります。 レポート結果の正確性に影響を与える可能性があることに留意してください。 プロジェクトチームからユーザーを削除するには、スタッフ/ユーザーに移動し、ユーザーを選択してから、リストの上に表示される「削除」ボタンをクリックします。

**質問**

テキストモード（グループ化）で列の降順を変更するにはどうすればよいですか？

**回答**

レポートの作成時に、「列（表示）」タブでほとんどの列を並べ替えることができます。 グループ化がない場合は、リストレポート全体がソートされます。 グループ化がある場合は、各グループ内の選択に従ってソートされます。

**質問**

承認ステージに割り当てられたチームメンバーを識別する列を作成するにはどうすればよいですか？

**回答**

タスクまたはイシュー/リクエストレポートを実行している場合は、Report Builder内に「承認者とステータス」という列があり、この情報が取り込まれます。
