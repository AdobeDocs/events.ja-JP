# Workfront Expert Insights - Evolution of an Instance with Jen Desmond

Expert Insightsの最新エピソードへようこそ！ あなたの「オンデマンド」Workfront CSMのCynthia Boonです。 他社のお客様がWorkfrontインスタンスを進化させる方法や、負担に感じることなく変化に適応する方法に疑問を抱くことはありませんか？

このエピソードでは、AdobeのテクニカルサクセスアドバイザーであるJen Desmond氏と共に、Workfrontを初めて利用する利用者から、信頼できるエキスパートへの道のりを解説し、変化を管理し、自社のインスタンスを成長させるための実践的なアドバイスを提供します。

>[!VIDEO](https://video.tv.adobe.com/v/3483111/?learn=on&enablevpops)

有益なインサイトを獲得できます。

* Workfrontの変化が「重い」と感じられる理由と、それをポジティブなマインドセットで再構築する方法
* 変化を乗り切る際に制御できることに焦点を当て、受け入れることの重要性
* 変化への適応にユーザーを巻き込む方法（変化が起こるのではなく）
* プロセスを簡素化し、進化すべきタイミングを把握するための実際の戦略
* インスタンスを最適化する際の労力と価値のバランスを取るという新たな視点

「負荷を軽くする方法」質問についてのより多くの考え：
これは、賛否両論があるかもしれませんが、導入のタイムラインを増やしましょう。 変更のために公開日を決め過ぎないようにしましょう。 計画では、必要なことをすべてカバーするために全力を尽くしますが、計画にはどれくらいの頻度で取り組むべきでしょうか？ 誰が苦しんでいるかを知っているからといって、いじめられたくはないのです。 ユーザー： そこからユーザーの導入タンクと幸運を祈ります。 関係者の同意を得て成約に導くことは難しいことも知っています。実際には、さまざまな議論を通じて、多くの時間を費やしたり、後で多額の費用を支払ったりすることになります。

もう一つは、コミュニケーションを忘れないでください！
変化に直面するとき、コミュニケーション、コミュニケーション、コミュニケーション！ 変更の具体的な全体像を把握したら、すぐにユーザーに通知します。 驚かさないようにし、それが起こるまで彼らを訓練するのを待ってはいけません。 悩ませる時間と余裕を与え、変化が生じる前にサポートし、準備を整えるようにしましょう。

## 講演者について

Jen Desmondは、シニアのテクニカルサクセスアドバイザーであり、顧客と連携して複雑なプロセスを簡素化し、導入を改善し、チームが本来の作業方法だけでなく、実際の作業方法を反映したソリューションを設計する必要があります。 Workfront Fusionに大きな情熱を注いでいます。自動化によって手作業が削減され、システムが連携し、プロセスを拡張する方法を自信を持って理解できるよう支援しています。 Jenは、組織がより迅速に行動し、より効果的にコラボレーションし、持続可能な方法で成長するのを支援するために、何が実用的であるのかに常に重点を置いています。
