# エキスパートへの質問 — ミックスと容量について

企業内での混在と容量を測定する方法を学びます。 本ウェビナーは 2019 年 10 月 2 日に録画されました。

>[!VIDEO](https://video.tv.adobe.com/v/341119/?quality=12)

## Q&amp;A

**質問**

%s を縦棒グラフに配置する方法

**回答**

グラフタブで「列をグループ化」と「100%に積み上げ」を選択したので、混合レポートに含まれる%値は存在しました。 「積み重ね」を選択した場合は、割合ではなく、予定時間の合計が表示されます。

**質問**

部門/組織のワークロードがプロジェクト/タスクと問題/リクエストの組み合わせの場合、WPI の (Workfrontレポートでの ) 高レベルのレビューを受け取る方法をお勧めします。

**回答**

プロジェクト、タスクおよび問題には、それぞれ独自のカスタムフォームを使用して独自のレポートを作成する必要があります。 ただし、それぞれ同じ「作業タイプ」フィールドを使用できます。 これら 3 つのレポートを 1 つのダッシュボードに表示できます。

**質問**

タスクを運用または戦略的にするために、タスクを一括編集する必要がありますか？

**回答**

私たちが提案している技術は、運用可能なタスクのリストを取得できるレポートを作成することです。 一度にすべてのタスクを選択し、一括編集して、作業用タイプのカスタムフォームを添付し、すべてのタスクの作業用タイプを一度に「運用可能」に設定します。 同じ手順に従って、戦略タスクのリストを収集し、それらを一括編集し、カスタムフォームを追加するなどします。

オンラインセミナでは、タスク名、プロジェクト所有者、ポートフォリオ、割り当てられたユーザーなどの特定の単語を確認するなど、カスタムプロンプトに関するいくつかのアイデアが説明されています。 これは単なるアイデアです あなたの状況で最も効果的なレポートを考え出すべきです。

**質問**

4 つのカテゴリが混在している場合、それぞれの目標を作成し、予測と計画の差分と実際の差分に関するレポートを作成することはできますか。 （4 つのカテゴリ、キャンペーン、ビジネスユニット、Web、製品）。 カスタムフォーム/フィールドのプロジェクトレベルにカテゴリがあります。 目標は、四半期ごとの予測（予算/予測）を作成し、その予測に向けた計画時間を追跡し、最終的に実績を計測することです。

**回答**

すべてのカテゴリが 1 つのカスタムフィールドに含まれている場合（現時点では、「作業タイプ」と呼びます）、最初に「計画時間」と「作業タイプ」に基づくプロジェクトレポートのグループを作成します。 プロジェクトレポートをフィルターして、Planning 内のプロジェクトを希望の日付範囲で表示します。 割合を表示する場合は、グループ化された列または棒グラフを 100%に積み重ねて使用します。 これは、予測レポートにする必要があります。

レポートをコピーして編集し、現在のプロジェクトに基づくレポートを作成し、計画時間に基づく組み合わせを表示できます。

レポートを再度コピーし、予定時間ではなく実際の時間でグループ化し、希望する日付範囲内の完了したプロジェクトのみを表示できます。 これにより、実際の時間に基づく混合率が表示されます。

**質問**

プロジェクトまたはタスクに複数のカテゴリ ID がある場合、この機能は使用できますか？

**回答**

はい。複数の ID がある場合、次のように、タブで区切る必要があります。

```
EXISTS:1:$$EXISTSMOD=NOTEXISTS
EXISTS:1:$$OBJCODE=OBJCAT
EXISTS:1:categoryID=5d76d82600e7926bb51eeb1e0886810e 5d54288d01034619f2eb2c74f6472f18
EXISTS:1:objID=FIELD:ID
```

これらをタブ文字で区切る最善の方法は、最初にビルダーでカテゴリのリストを作成することです。 複数のカスタムフォーム名を入力し、テキストモードに切り替えると、タブで区切られた ID として表示されます。

複数の ID は OR として扱われるので、タスクにいずれかの ID が添付されている場合、レポートには表示されません。

**質問**

レポートに「ANDed」と表示されるか、「ORed」と表示されるか。

**回答**

個々のカスタムプロンプトは「AND」です。 そのため、プロジェクトの所有者として Pam を指定し、タスクに割り当てられた Bill を指定した場合は、Pam がプロジェクトの所有者であるプロジェクト内の Bill に割り当てられたタスクのみが表示されます。

**質問**

なぜ特定の列でしか並べ替えられないのですか？ 例えば、割り当てで並べ替えることはできません。

**回答**

「割り当て」はリストであり、項目のリストで並べ替えやグループ化を行うことはできません。 並べ替えやグループ化が可能なのは、個々の項目に対してのみです。

例えば、1 つのタスクで、次のような割り当てのリストが表示されるとします。

```
Jane
Bill
Dan
```

別のタスクに対する割り当てのリスト：

```
Bill
Jane
Helen
```

並べ替えで最初に表示するタスクはどれですか？ 「リスト内の名で並べ替え」と言っても、順序を制御できないので、これは必ずしも便利ではありません。 Workfrontは、リストでの並べ替えをまったく許可しないことで、問題を回避します。

リストでのグループ化について 名前のリストでグループ化できる場合は、ジェーン、ビル、ダンに割り当てられたすべてのタスクがグループ化され、ジェーン、ダン、ビル（同じリストですが、異なる順序で）に割り当てられたすべてのタスクが異なるグループに表示されます。 繰り返しますが、Workfrontでは、リストによるグループ化を許可しないことで、問題を回避します。

**質問**

計画的なタスクに予定時間を使用し、運用に実際の時間を使用しますか？

**回答**

いいえ。この例では、計画時間を使用して、戦略タスクと運用タスクの両方に対して計画された作業のレベルを示しています。 これにより、過去、現在、未来を問わず、簡単に比較できます。 また、実際の時間を使用して戦略タスクと運用タスクを比較することもできますが、実際の時間は、実際にタスクに費やした時間として人が報告した時間なので、過去のタスクに対してのみ使用できます。

**質問**

リソースプランナーでは、過去に計画されたが完了していないタスクをどのように計算しますか。 予定時間は進行しないので、リソースを必要とするタスク/時間として将来の週には表示されません。

**回答**

未完了の作業を「自動」でロールフォワードする機能はありません。 この処理が行われると、プロジェクトを再開する必要があります。 最初に特定のタスクに割り当てたリソースが新しい期間で使用できない場合は、プロジェクトマネージャーがこれを確認し、再生する最適な方法を決定する必要があります。 関係者が関与し、計画の変更に対する承認を得ることを意味する場合があります。

**質問**

メールや休憩のチェックに 2 時間/日入力するのではなく、FTE を調整するのがお勧めですか。

**回答**

はい、FTE を 0.75 に設定した場合、リソース・プランナで 1 日に 6 時間使用可能としてユーザーを表示します。 これは、毎日利用可能になる予定です。 各四半期の最終日を利用できないなど、日付に応じて異なる期間で使用不可と表示する場合は、オーバーヘッドプロジェクトを使用します。

自分で作成し、必要に応じて変更できるのに対し、自分の FTE を編集する権限がないのに対して、オーバーヘッドプロジェクトを好む人もいます。

**質問**

チームキャパシティダッシュボードのデータは、作業に対する権限に関係なく、レポートを共有しているすべてのユーザーが利用できますか？

**回答**

ユーザーがオブジェクトを表示する権限を持っていない場合、そのオブジェクトはレポート/ダッシュボードに表示されません。 ただし、全員に同じ結果を表示させたい場合は、レポートの操作/編集/レポート設定に移動し、「次のアクセス権を持つレポートを実行する」フィールドに自分の名前を入力します。 これにより、このレポートで共有されているユーザーが、表示されている正確な結果を表示できます。 結果自体への追加のアクセス権は付与されないので、一部の結果はクリックできない場合もあります。

**質問**

プロジェクト全体に割り当てられている（タスクレベルではない）すべてのスタッフを表示するレポートを作成するにはどうすればよいですか？

**回答**

プロジェクトレポート内で列を作成し、「スタッフ」タブ（プロジェクトチーム）の一部として表示されるすべてのユーザーを表示できます。 次のテキストモードを使用します。

```
displayname=Project Team
listdelimiter=<p>
listmethod=nested(projectUsers).lists
textmode=true
type=iterate
valuefield=user:name
valueformat=HTML
```

**質問**

チームの仕組みを組み込んだレポート/ダッシュボードを作成したいのですが、 特に、以下のシナリオ： — 自分が所有するプロジェクト/自分用に作成されたプロジェクト/他のユーザーに割り当てたタスク/自分に割り当てられたタスク

**回答**

自分が所有するプロジェクト

「現在のプロジェクト」という名前の組み込みレポートがあり、現在のプロジェクトがすべて表示されます。 このプロジェクトを編集し、フィルタールールを追加できます。プロジェクト/所有者 ID/次と等しい > $$USER.IDT 次に、レポートを保存して「所有するプロジェクト」に名前を変更します。

自分用に作成されたプロジェクト

「マイプロジェクト」という名前の組み込みレポートがあり、プロジェクトチームに参加している現在のプロジェクト（所有者、スポンサー、またはタスクに割り当てられている）がすべて表示されます。 これがあなたが求めているものかどうかは不明ですが、あなたをプロジェクトの所有者、スポンサー、またはタスクに割り当てる以外に、誰かがあなたのためにプロジェクトを作成したかどうかを知る方法は他にありません。

他のユーザーに割り当てたタスク

任意のフィルターでタスクレポートを作成し、「フィルター」タブに移動して、「テキストモードに切り替え」をクリックします。 既に存在するものにこのコードを追加します。

```
EXISTS:1:$$OBJCODE=ASSGN
EXISTS:1:taskID=FIELD:ID
EXISTS:1:assignedByID=$$USER.ID
```

ログインしたユーザーが現在の担当者の 1 つ以上を割り当てたすべてのタスクが表示されます。 担当者が複数の担当者によって割り当てられた場合は、担当者を最初に割り当てた人の名前のみがタスクのランディングページに「担当者」と表示されます。 割り当て済みの担当者と各担当者を割り当て済みの担当者をすべて表示する場合は、ビューに列を追加し、テキストモードに切り替えて、以下を使用します。

```
displayname=All Assignees and Requesters
listdelimiter=<p>
listmethod=nested(assignments).lists
textmode=true
type=iterate
valueexpression=CONCAT("Assigned To: ",{assignedTo}.{name},"; Requested By: ",{assignedBy}.{name})
valueformat=HTML
```

自分に割り当てられたタスク

「マイタスク」という名前の組み込みレポートがあり、タスクの所有者である現在のプロジェクトに関する未完了のタスクがすべて表示されます。 フィルターを変更して、多数のユーザーの 1 人として割り当てられている可能性のあるタスクを、タスクの所有者だけでなく、すべて表示するようにすることをお勧めします。 これをおこなうには、フィルタールールを削除します

```
Task > Assigned To ID > Equal > $$USER.ID 
```

そして置き換えて

```
Assignment Users > ID > Equal > $$USER.ID
```

**質問**

グラフのラベルをカスタマイズする方法はありますか？ プロジェクトのステータスを反映するグラフを作成すると、ホームグループの名前がラベルに含まれることに気づきました。

**回答**

グラフのラベルは、グループ化するフィールド名を使用します。 したがって、ラベルを変更できる唯一の方法は、任意の名前の計算済みカスタムフィールドを使用することです。 「 」フィールドの計算セクションで、グループ化の基準となるネイティブフィールドのフィールド名を入力します。

**質問**

チャックのチーム担当者のレポートバーの色を変更する方法を教えてください。 例えば、後ろは琥珀色、遅くは赤色、時間には緑色ですか。 また、順序をより論理的に変更することもできます（赤/琥珀/緑、逆）。

**回答**

[ 進捗状況ステータス ] オプションのレポートで使用する色を変更するには、レポートを編集し、[ グラフ ] タブをクリックします。 「カスタムカラー >」ドロップダウンを探します。 列またはバーをグループ化するかどうかに応じて、「左 (Y) 軸」ボックスまたは「データをグループ化」ボックスの横に表示されます。 このドロップダウンで、色を選択できます。 色オプションの右下にある数字をクリックすると、大きなパレットから色を選択できます。

残念ながら、これらの順序を変更することはできません。

**質問**

ワーカーにタスクが割り当てられているプロジェクトの数を示すグラフを作成できますか？

**回答**

はい、その方法は次のとおりです。

* プロジェクトレポートの作成
* 問題のユーザーがログインユーザーの場合、フィルターには次の行を含める必要があります。

```
   Project Users > ID > Equal >$$USER.ID 
```

* そうでない場合は、次の場所にユーザー名を配置します。 [!DNL $$USER.ID]. これにより、この人にタスクが割り当てられている、または所有者またはスポンサーであるすべてのプロジェクトが表示されます。 タスクが割り当てられたプロジェクトのみを表示する場合は、次の 2 つの追加のフィルタールールを追加する必要があります。

```
   Project > Owner ID > Not Equal > $$USERID
   Project > Sponsor ID > Not Equal > $$USERID
```

* グラフを作成できるように、少なくとも 1 つのグループを作成します。 会社など、あらゆるものに対してグループ化します。 次に、「グラフ」タブをクリックし、グラフを選択します。 1 つの軸のデフォルトは「レコード数」です。 これは、ユーザーが割り当てをおこなったプロジェクトの数です。

ユーザーにプロジェクト（タスクまたはプロジェクト所有者またはプロジェクトスポンサーに割り当てられた）の割り当てが与えられると、その担当者がプロジェクトチームに追加され、「人」サブタブの「スタッフ」タブに表示されます。 ユーザーがプロジェクト所有者、スポンサー、またはタスクの割り当てから削除された場合、その名前はプロジェクトチームに残ります。 削除したい場合は、手動で削除する必要があります。 これは、レポート結果の精度に影響を与える可能性があることに注意してください。 プロジェクトチームから他のユーザーを削除するには、「Staffing」>「People」に移動し、1 人または複数のユーザーを選択して、リストの上に表示される「削除」ボタンをクリックします。

**質問**

（グループ化で）テキストモードで列の降順を変更するにはどうすればよいですか？

**回答**

レポートを作成する際に、「列（表示） 」タブでほとんどの列の並べ替えを選択できます。 グループ化がない場合、リストレポート全体が並べ替えられます。 グループ化がある場合、各グループ内での選択に従って並べ替えられます。

**質問**

承認ステージに割り当てられたチームメンバーを識別する列を作成するにはどうすればよいですか？

**回答**

タスクまたは問題/要求レポートを実行している場合は、Report Builder内に「承認者とステータス」という列が表示され、この情報が取り込まれます。
