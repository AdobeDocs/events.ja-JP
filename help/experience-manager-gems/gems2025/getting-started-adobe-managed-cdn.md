# AEM GEM - Adobe Managed CDNの概要

AEM Cloud ServiceのAdobe Managed CDNとその設定方法について説明します。 AEM as a Cloud Service アプリケーションのパフォーマンスとセキュリティの両方を向上させるために使用できる、新しいCDN設定機能についてご紹介します。 このセッションでは，

* Adobe CDNとは
* AEMaaCSおよびEdge Delivery Servicesに関連するトポロジ
* CDN ルールで実装できる一般的なユースケース
* RDEを使用してCDN設定を迅速にテストおよびデプロイする方法

## 重要な留意点

### Adobe Managed CDNの主な機能

* **カスタムドメインと証明書** カスタムドメインと証明書をホストして安全な接続を確立するために必須です。
* **キャッシュ** キャッシュからのHTTP レスポンスの配信は、オリジンからの取得（数百ミリ秒未満）と比較して、大幅に高速です（10 ミリ秒未満）。
* **すぐに利用できるカスタム CDN** Adobeでは、すぐに利用できる管理CDNを提供していますが、ユーザーは独自のCDNを持ち込むこともできます。

### 設定オプション

* **変換を要求** ヘッダーの変更、パスの書き換え、トラフィックのブロック、およびリクエストのリダイレクト。
* **トラフィックフィルター**&#x200B;特定のルールに基づいてトラフィックをブロックまたは許可します。
* **認証** エッジキー、パンチキー、基本認証のサポート。
* **オリジン セレクター**&#x200B;定義されたルールに基づいて、異なるオリジンにプロキシ要求を送信します。
* **応答変換**&#x200B;応答ヘッダーとステータスを変更します。

### 実装とカスタマイズ

* **設定パイプライン** YAML ファイルをデプロイしてCDN ルールを設定します。
* **トラフィック保護** トラフィックフィルタールールを使用して、パターンに基づいてトラフィックをブロック、記録、警告します。
* **レート制限** IPあたりのリクエスト数を制限することで、DDoS攻撃から保護します。

### ツールと分析

* **Elasticsearch Kibana Stack**&#x200B;提供されたダッシュボードで利用状況とトラフィックを分析します。
* **ログ転送**&#x200B;分析のためにSplunk インスタンスにログを転送します。

### デモのハイライト

* **設定のデプロイ** トラフィックフィルタールールとリダイレクトのデプロイのデモ。
* **認証とオリジンの選択**&#x200B;は、基本認証とプロキシトラフィックを異なるオリジンに設定する方法を示しました。

### ベストプラクティス

* **高速レスポンス**&#x200B;脆弱性を回避するために、オリジンからの高速レスポンスを確保します。
* **優れたキャッシュ** キャッシュを活用してトラフィックを効率的に処理します。
* **ダッシュボードを使用** トラフィックと使用状況を分析して、適切なレート制限を設定します。
* **戦略を組み合わせる**&#x200B;別のレート制限戦略を使用して保護を強化します。
* **アラートを設定** サイトが攻撃を受けたときに通知を受け取ります。
* **ルールをテスト** ルールが期待どおりに動作することを確認するために、ブロックする前にログアクションを開始します。
