# API ツアー – 過去6か月間に出荷した新しいAPIを確認する

API ツアー – 過去6か月間に出荷した新しいAPIを確認する

AEMのヘッドレスアーキテクチャへの移行について説明します。 新しい焦点は、ユーザーがシンプルで強力なAPI呼び出しを通じてすべてのアクションを実行できるようにしながら、プロセスのスピードを念頭に置いて行うことです。 説明する2つのAPIは、コンテンツフラグメントモデル（CFM）とコンテンツフラグメント（CF）コントロールプレーン APIです。 これらのAPIを利用することで、複雑なモデルを容易に作成し、すばやく容易にデータを入力できます。 新しいコンテンツフラグメントモデルを作成し、それに基づいてコンテンツフラグメントを入力する方法について説明します。 次に、モデルを更新し、新しい構造に基づいて他のフラグメントを作成します。 「初期の部分が廃止されたため、単一のリクエストでより多くのコンテンツフラグメントモデルを作成することを中心に、より複雑なユースケースを検討し始めます。 →次の場所で会話を続けます：https://adobe.ly/3oXY0gV
