# Microsoft 365/Google Workspaceを使用した優れたエディター体験の開発

Microsoft 365/Google Workspaceを使用した優れたエディター体験の開発

各プロジェクトの重要な側面は、コンテンツドキュメントの構造です。 作成者は、問題なく直感的にドキュメントをナビゲートできる必要があります。 コンテンツドキュメントの構成方法に関するガイドラインとヒントとコツを提供します。 Experience Manager用のドキュメントを作成する場合、コンテンツを提供する作成者がシンプルかつ直観的に作成できることが不可欠です。 このセッションでは、コンテンツ作成者がコンテンツ構造を操作できるように、コンテンツファーストのアプローチでコンテンツドキュメントを構成する方法を説明します。 ベストプラクティス、セクションやブロックを使用する方法とタイミングについて説明し、プロジェクトで陥りやすい落とし穴を紹介します。 →次の場所で会話を続けます：https://adobe.ly/40YBTnO
