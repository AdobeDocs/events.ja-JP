# アセットセレクターを介したオープン APIを備えたDynamic Mediaを使用した、スケーラブルなWebに最適化された配信とコンテンツ変換

Adobe Experience Manager Assets as a Cloud ServiceとDynamic Mediaが、コンテンツ配信をどのように改善するのかをご確認ください。 エンジニアリングマネージャーのKaranjeet Singh氏は、スマート切り抜き、プリセットオーサリング、Edge Delivery Servicesやサードパーティアプリケーションとの統合を通じて、ワークフローを合理化する方法を紹介します。

## 重要な留意点

* **オープン APIを使用したDynamic Mediaの概要**
   * リッチメディアアセット（画像、動画）をシームレスに管理、配信し、カスタマイズと最適化を即座に実行できます。
   * さまざまなアプリケーション（Adobeおよびサードパーティ）向けのassets as a cloud serviceの民主化。

* **利点**
   * バイナリコピーは不要で、アセットのライフサイクルは維持されます。
   * パブリックおよび制限付きアセット配信URL。
   * オープン APIによる、開発者にも使いやすいプログラマティック管理。

* **統合パターン**
   * アセットセレクター経由で簡単に共有。
   * 検索オープン API経由で接続されています。

* **コンテンツの変換とWebに最適化された配信**
   * 異なる目的のための4種類のURL （レンディション、コンテンツ変換、アダプティブビデオストリーミング、メタデータ）。
   * 新しい高度な修飾子（プリセット、レイヤリングマスク、DPR アンシャープマスク、スマートイメージング）が導入されました。
   * AVおよびTiff出力フォーマットのサポート。

* **スマートイメージング** ブラウザー形式の変換、ネットワーク帯域幅の最適化、デバイスピクセルレシオ（DPR）です。

* **アセットセレクター**
   * Adobe Experience Manager Assetsを統合するためのユーザーインターフェイス。
   * 2つのUI フレーバー：モーダルとリアルビュー。
   * 下流アプリケーションでの統合手順と使用例。

* **例とデモ**
   * カスタムメールクライアントとサイトページの統合：
   * 高度な修飾子とスマート切り抜きの使用。
   * アダプティブオーディオストリーミングとホットスポットの統合による配信サービス。
