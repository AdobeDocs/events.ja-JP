# Commerceストアフロントの最適化

AdobeのカスタマーテクニカルアドバイザーであるAndrii Abumuslimov氏が、Commerceインスタンスの最適化に関するライブウェビナーをリードしています。 定期的なケイデンスのメンテナンスから、一般的なデータブロッカーに対する予防策まで、ストアフロントを効率的に運営するために知っておくべきことを包括的に解説しています。 これには、ボットのアクティビティを追跡および軽減するための戦略や、ディスク容量とデータベースサイズの効果的な管理と計画が含まれます。

>[!VIDEO](https://video.tv.adobe.com/v/3443031/?learn=on&enablevpops)

*発表者*

* Adobeのデジタルエンゲージメントストラテジストである&#x200B;**Jeff McGuire**
* **Andre**&#x200B;は、Adobe Commerce （旧Magento）を専門とするAdobeのカスタマーテクニカルアドバイザーです

## 重要な留意点

このウェビナーは、e コマースのストアフロントを最適化することに重点を置いています。
インタラクティブで、最後にQ&amp;A セッションがあります。
セッションは録画され、オンデマンドで利用できます。

* **対象トピック**

   * ストレージ使用状況の監視と最適化：
   * webのweb クローラー、トラフィックの検出、軽減：

* **ストレージ最適化**

   * ストレージの使用状況のモニタリングには、New Relicを使用します。
   * データ管理と最適化のベストプラクティスを実装する。
   * ディープイメージ最適化を有効にして、ディスク容量を節約します。

* **Web Web クローラー**

   * New Relicやカスタム SQL クエリなどのツールを使用して、不正なwebweb クローラーを検出および低減。
   * 非正規ボットのレート制限およびブロック戦略を実装します。
   * 高度なボット検出には、Human Bot Defenderなどの高度なサービスを検討してください。

* **繁忙期の準備**

   * 事前に負荷テストを実施します。
   * ボットトラフィックを最適化および軽減し、販売量の多い時期にスムーズに運用できるようにします。
   * 必要に応じて、メンテナンスとハードウェアのアップグレードをスケジュールします。

* **リソースとフォローアップ**

   * ホワイトペーパーとその他のリソースへのアクセス
   * ウェビナーの録画はメールでお送りします。
